## Overview

Generates a professional, self-contained HTML design system document from extracted brand tokens, classified images, and brand profile data. The output is a deliverable-quality HTML file with all CSS inlined — no external dependencies, no CDN links, no JavaScript. Used as the final reference document in the website pipeline: shows the creative team and the customer exactly what brand assets, colors, typography, and imagery are available before any site is built.

## Steps

1. Read inputs (use what is available; skip gracefully if a source is absent):
   - `design-tokens.json` (W3C Design Token format): `color`, `typography`, `spacing`, `border-radius` token groups
   - `image-library.yaml`: classified images from `image-extract` Pass 2
   - `brand-profile.yaml`: brand name, tagline, style keywords, primary/secondary colors

2. Build the HTML document structure (all CSS in a `<style>` block in `<head>`):
   - Root CSS variables derived from design tokens (e.g., `--color-primary`, `--font-heading`)
   - Page max-width: 1200px, centered, generous whitespace
   - Sans-serif system font for document UI; brand fonts (if available) for the Typography demo section only
   - Neutral background (`#fafafa`), white section cards, subtle `box-shadow`

3. **Section 1 — Color Palette**
   - Heading: "Color Palette"
   - For each color token: render a swatch `div` (60×60px rounded square) filled with the color value
   - Below each swatch: token name, hex value, RGB equivalent
   - Group by token category if present (e.g., Primary, Secondary, Neutral, Surface)
   - If no color tokens: render a placeholder card "No color tokens extracted"

4. **Section 2 — Typography**
   - Heading: "Typography"
   - For each font family found in tokens: display a sample block showing:
     - Font family name as the label
     - Sample text at h1 (2.5rem), h2 (2rem), h3 (1.5rem), body (1rem), caption (0.875rem) sizes
     - Sample text: "The quick brown fox jumps over the lazy dog" for each size
   - Show font-weight variants if multiple weights are present in tokens
   - If no typography tokens: use `font-family: system-ui, sans-serif` with a note "System font (no brand fonts extracted)"

5. **Section 3 — Image Library**
   - Heading: "Image Library"
   - Grid layout: 3 columns on wide, 2 on narrow (use CSS Grid with `auto-fill`)
   - For each image in `image-library.yaml`:
     - `<img>` tag with `src` pointing to `local_path`
     - Below image: classification badge (pill with category name), alt text, quality_score formatted as percentage
     - Classification badge colors: hero=indigo, headshot=teal, logo=amber, product=violet, event=green, background=slate, decorative=gray
   - If no images: render placeholder card "No images extracted from website"

6. **Section 4 — Component Samples**
   - Heading: "Component Samples"
   - Render 4 component previews using brand colors and fonts:
     - **Button**: primary and secondary variants. Primary: brand primary color background, white text. Secondary: outline style.
     - **Card**: white card with subtle shadow, heading, body text, and a CTA link. Uses brand typography.
     - **Navigation bar**: horizontal nav with brand name on left, 4 placeholder nav links. Brand primary color for active state.
     - **Footer**: dark background (brand primary or `#1a1a1a`), brand name, 3 column links, copyright line.
   - Wrap each in a labeled preview frame

7. **Section 5 — Logo**
   - Heading: "Logo"
   - If an image classified as `logo` exists in `image-library.yaml`: display it at natural size on a white background AND on a dark background to show versatility
   - If no logo found: render placeholder card "No logo extracted — upload logo separately"

8. Write the complete HTML to `output/design-system.html`.
   - Ensure all file references use relative paths matching the pipeline output directory structure
   - Add a document header with the brand name, generation date, and "Generated by OpenClaw Website Pipeline"

## Self-Containment Rules

- All CSS must be inside a single `<style>` block in `<head>` — no `<link>` tags to external stylesheets
- No `<script>` tags of any kind
- No Google Fonts CDN or any external URL (if fonts are needed, use `system-ui` fallback)
- Images referenced via relative `src` paths only — no data URIs (files are already downloaded)
- The HTML file must render correctly when opened directly in a browser from the filesystem

## Graceful Degradation

| Missing input | Behavior |
|---|---|
| No `design-tokens.json` | Derive approximate colors from `brand-profile.yaml`; note "Tokens not extracted" in document header |
| No `image-library.yaml` or empty entries | Omit Section 3 and Section 5 entirely; do not render empty grids |
| No `brand-profile.yaml` | Use brand name "Your Brand", neutral colors (`#333`, `#666`), system fonts |
| No logo | Render placeholder text in Section 5 |

## Output

- `output/design-system.html` — standalone HTML design system document, professional deliverable quality

## Error Handling

- Malformed JSON in `design-tokens.json`: skip the token file, derive from brand-profile, continue
- Invalid image paths in `image-library.yaml`: omit that image from the grid, continue with others
- Empty inputs across all sources: generate a minimal document with placeholder content and a warning banner at the top

## Requirements

- Filesystem: `write` (writes output/design-system.html)
- Network: `false` (pure HTML generation, no external calls)
- No binary dependencies
- No env vars required
